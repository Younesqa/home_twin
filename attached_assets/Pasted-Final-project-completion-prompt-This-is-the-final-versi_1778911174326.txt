PART 4: SOLAR ENERGY PAGE
==================================================

Make sure there is a protected page:
/solar

Arabic title:
"الطاقة الشمسية"

English title:
"Solar Energy"

Show Solar Energy link in Header ONLY if:
setup?.hasSolar === true

If user hasSolar is false:
- Do not show Solar link.
- If user opens /solar manually, show:
Arabic: "هذه الصفحة متاحة فقط للمستخدمين الذين لديهم خلايا شمسية."
English: "This page is only available for users who have solar panels."

Solar simulation rules:
- generation_per_minute = 0.05 kWh
- municipality buying rate = 0.60 NIS per kWh
- available energy increases over time
- user can sell available energy to municipality
- earned money is added immediately to wallet

If needed, create solar_accounts table safely:

CREATE TABLE IF NOT EXISTS solar_accounts (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL UNIQUE,
  available_kwh REAL NOT NULL DEFAULT 0,
  rate_per_kwh REAL NOT NULL DEFAULT 0.60,
  generation_per_minute REAL NOT NULL DEFAULT 0.05,
  last_updated_at TEXT NOT NULL,
  created_at TEXT NOT NULL,
  FOREIGN KEY(user_id) REFERENCES users(id)
);

When setup hasSolar = true:
- create solar account if missing.

When setup hasSolar = false:
- do not create account.
- do not delete old solar data.
- only hide page/link.

Solar API:
GET /api/solar/summary
- If no solar, return hasSolar false.
- If has solar:
  - ensure account exists
  - update available_kwh based on elapsed time
  - return availableKwh, ratePerKwh, generationPerMinute, walletBalance

POST /api/solar/sell
Body:
{ kwh: number }

Validation:
- kwh > 0
- kwh <= availableKwh
- user must have solar

Logic:
- update available energy first
- earnings = kwh * ratePerKwh
- subtract sold kWh
- add earnings to wallet
- create wallet transaction:
  type = solar_sale
  description = "بيع طاقة شمسية للبلدية"
- return updated summary
