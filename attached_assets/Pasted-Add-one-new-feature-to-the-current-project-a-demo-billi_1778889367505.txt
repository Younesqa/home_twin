PART 4: DATABASE TABLES
==================================================

Update SQLite database.

Add these tables if they do not exist:

1. wallets

CREATE TABLE IF NOT EXISTS wallets (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL UNIQUE,
  balance REAL NOT NULL DEFAULT 0,
  updated_at TEXT NOT NULL,
  FOREIGN KEY(user_id) REFERENCES users(id)
);

2. invoices

CREATE TABLE IF NOT EXISTS invoices (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL,
  title TEXT NOT NULL,
  amount REAL NOT NULL,
  status TEXT NOT NULL DEFAULT 'unpaid',
  type TEXT NOT NULL DEFAULT 'current',
  month TEXT NOT NULL,
  created_at TEXT NOT NULL,
  paid_at TEXT,
  FOREIGN KEY(user_id) REFERENCES users(id)
);

Invoice status:
- unpaid
- paid

Invoice type:
- current
- previous

3. wallet_transactions

CREATE TABLE IF NOT EXISTS wallet_transactions (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL,
  type TEXT NOT NULL,
  amount REAL NOT NULL,
  description TEXT NOT NULL,
  created_at TEXT NOT NULL,
  FOREIGN KEY(user_id) REFERENCES users(id)
);

Transaction type:
- topup
- bill_payment

Important:
Each citizen must have one wallet.
Create wallet automatically after registration if it does not exist.

Protected citizen routes:

Returns:

    balance
  },
  currentInvoice,
  previousInvoices,
  transactions
}

currentInvoice:
latest unpaid or current invoice.

previousInvoices:
all old invoices ordered newest first.

transactions:
latest wallet transactions ordered newest first.

POST /api/billing/topup

Body:
{
  amount,
  cardholderName,
  cardNumber,
  expiry,
  cvv
}

Demo validation:
- amount must be positive
- cardholderName required
- cardNumber required but do not store it
- expiry required but do not store it
- cvv required but do not store it

Important:
Do NOT store cardNumber.
Do NOT store cvv.
Do NOT store expiry.
Only simulate payment.

Always succeed if fields are filled.

Logic:
- Add amount to wallet balance.
- Insert transaction:
type = topup
amount = amount
description = "إضافة رصيد" / "Wallet top-up"
- Add activity log:
Arabic: "تم إضافة رصيد إلى المحفظة"
English: "Wallet balance added"
- Return updated billing summary.

POST /api/billing/pay-invoice/:invoiceId

Logic:
- Get invoice for current user.
- If not found, return error.
- If already paid, return error.
- Get wallet.
- If wallet balance < invoice amount:
  return error:
  Arabic: "الرصيد غير كافٍ لدفع الفاتورة"
  English: "Insufficient balance to pay the bill"

If enough balance:
- Subtract invoice amount from wallet.
- Set invoice status = paid.
- Set paid_at = now.
- Insert transaction:
type = bill_payment
amount = invoice.amount
description = "دفع فاتورة الكهرباء" / "Electricity bill payment"
- Add activity log:
Arabic: "تم دفع الفاتورة بنجاح"
English: "Bill paid successfully"
- Return updated billing summary.
